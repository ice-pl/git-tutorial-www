Tworzenie repozytorium
  1. Tworzymy nowy folder
    mkdir nowy
  2. wchodzimy do tego folderu
    cd nowy
  3. inicjalizujemy repozytorium
    git init
  4. wchodzimy do folderu poprzez bash git-a
git init
Dodawanie do kolejki
  1. sprawdzamy bieżący status
    git status
  2. tworyzmy pliki
    echo >> nowy.txt
    echo >> do_ukrycia.txt
  3. dodajemy pliki do kolejki
    git add .
  4. sprawdzamy bieżący status
    git status
  5. wprowadzamy zmiany do pliku i sprawdzamy bieżący status
    git status
    mamy dwie wersje pliku i musimy zdecydować co zrobić: usunąć którąś kopię, przywrócić oczekującą na zatwierdzenie lub ponownie dodać plik do kolejki oczekiwania
    na czerwono - pliki nie śledzone
    na zielono - pliki śledzone
  6. tworzymy commit
    git commit -m "Pierwsza rewizja"
git status git add . git commit -m "Pierwsza rewizja"
Ignorowanie plików
  1. tworzymy plik z listą ignorowanych plików i folderów
    nano .gitignore
  2. w edytorze nano wpisujemy ignorowane pliki / foldery
  3. sprawdzamy bieżący status
    git status
    brakuje pliku do_ukrycia.txt, a pojawił się .gitignore
  4. usuwamy .gitignore
    rm .gitignore
  5. sprawdzamy bieżący status
    git status
    plik do_ukrycia.txt jest widoczny
nano .gitignore
Modyfikowanie kolejki
  1. dodajemy pliki do kolejki
    git add .
  2. sprawdzamy bieżący status
    git status
  3. resetujemy kolejkę
    git reset
  4. sprawdzamy bieżący status
    git status
    po resecie kolejki wszystkie pliki, które w niej były stają się nie śledzone
  5. dodajemy pliki do kolejki
    git add .
  6. sprawdzamy bieżący status
    git status
  7. usuwamy z kolejki określony plik / pliki
    git rm --cached do_ukrycia.txt
  8. sprawdzamy bieżący status
    git status
    po usunięciu pliku z cache-u plik, który został usunięty staje się nie śledzony, a pozostałe nadal są śledzone
git add . git status git reset git rm --cached do_ukrycia.txt
